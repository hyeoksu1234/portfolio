01

Problem

어떤 문제를 발견했는지

조직의 정렬 상태는 감으로만 이야기되고 있었습니다.

리더가 전달했다고 생각하는 전략과 팀원이 실제로 이해한 내용의 간극을 객관적으로 확인하기 어려웠고, 진단 이후 행동이 실제 성과로 연결됐는지도 추적되지 않았습니다.

  • 전략 이해도와 네트워크 복잡도를 함께 볼 수 있는 진단 구조가 필요했습니다.
  • AI 분석 실패가 전체 진단 실패로 이어지지 않는 복원력이 필요했습니다.
  • 웹과 Electron에서 서로 다른 인증 저장 방식을 안전하게 다뤄야 했습니다.

02

Decision

무엇을 판단했고 왜 그렇게 했는지

01

진단 결과를 정렬 매트릭스와 네트워크로 이중 시각화

평균 점수만으로는 팀 내부의 연결 구조와 병목을 설명하기 어려웠기 때문입니다.

02

3개 LLM 공급자 자동 폴백

한 공급자의 장애·제한이 분석 전체를 중단시키지 않도록 Anthropic–OpenAI–DeepSeek 순으로 복구 경로를 설계했습니다.

03

행동–성과–패턴–규칙 피드백 루프

한 번의 진단 보고서에서 끝나지 않고 실행 효과를 학습하는 제품으로 만들기 위해서입니다.

03

Process

어떤 과정을 거쳤는지

  1. 01

    가설 정의

    조직 정렬을 전략 이해, 관계망, 실행 결과로 나누고 측정 가능한 데이터 구조로 번역했습니다.

  2. 02

    제품화

    대시보드, 정렬 매트릭스, Canvas 네트워크 시각화와 인증·권한 구조를 구현했습니다.

  3. 03

    업무 도구 연결

    Slack 슬래시 커맨드와 모달 수집 흐름, Electron 데스크톱 경험을 연결했습니다.

  4. 04

    검증 과제 분리

    구현 규모와 고객 성과를 구분하고, 실제 고객 전후 KPI가 확보되기 전에는 제품 성과로 주장하지 않기로 했습니다.

04

Outcome
